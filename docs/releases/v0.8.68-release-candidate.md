## Verification and publication gates

Candidate metadata alignment and Cargo publish/package dry-runs have passed in
the current audit. Before calling v0.8.68 ready or published, complete all of
the following against the final commit:

1. Run formatting, targeted regression tests, workspace/release checks, and a
   locked release build; record any platform or provider limitation.
2. Install the final local binaries and manually exercise the underwater shell,
   settings/permission controls, links, composer, worker transcripts, Workflow
   controls, cancellation, and reduced-motion paths.
3. Push the candidate branch, open and review a PR against `main`, and update
   the issue accounting above from reviewed evidence.
4. Merge to `main`, create signed/verified `v0.8.68`, and let the release
   workflow produce and verify checksummed platform assets.
5. Publish the ordered `codewhale-*` crate set and npm `codewhale@0.8.68`, then
   rerun `scripts/release/check-published.sh 0.8.68` until it passes.
6. Confirm the GitHub release, updater/install paths, Cargo installs, npm
   wrapper, and package registries all resolve the same final build.

Do not install or publish the local experimental Z.AI plugin draft as part of
v0.8.68; it is outside this candidate and has not passed the release gates.
